# TikTok 广告投放系统 V2 需求文档

> 以投放师日常工作流为核心，按「看全局 → 发现异常 → 执行动作 → 验证效果」闭环设计

---

## 一、看全局 —— 「一屏掌控今日战况」

### 1.1 今日大盘（现有 Dashboard 增强）

**现状**：Dashboard 有基础指标但缺乏周期对比和趋势感知。

**需求**：

| 指标卡片 | 当前值 | 对比维度 | 说明 |
|---------|--------|---------|------|
| 今日花费 | $XX.XX | vs 昨日同时段 / vs 上周同天 | 颜色标注涨跌 |
| 今日 GMV | $XX.XX | vs 昨日 / vs 7日均值 | |
| 今日 ROAS | X.XX | vs 昨日 / vs 保本线 | 低于保本线红色警告 |
| 今日订单数 | XX | vs 昨日 / vs 7日均值 | |
| 今日真实利润 | $XX.XX | vs 昨日 | 需成本配置 |
| 预算消耗率 | XX% | 当前花费/日预算 | 超 80% 提示 |
| 花费速率 | $X.XX/h | vs 昨日同时段速率 | 判断是否烧得太快 |

**实现要点**：
- 后端新增 `/dashboard/realtime` 接口，查 `metrics_snapshots` 今日 vs 昨日同 `stat_date` 数据
- 前端指标卡片加 `↑12%` / `↓5%` 标签，绿涨红跌
- 保本 ROAS 从 `product_costs` 聚合计算

### 1.2 多店铺对比看板

**现状**：店铺页面只展示单店数据，无法横向对比。

**需求**：
- 多店铺同屏：每个店铺一个卡片，核心指标并排
- 按 ROAS / 花费 / GMV 排序
- 支持选中 2-3 个店铺做趋势对比图

### 1.3 Campaign 矩阵视图

**现状**：Campaign 列表只有表格形式。

**需求**：
- 矩阵气泡图：X 轴=花费，Y 轴=ROAS，气泡大小=GMV
- 一眼看出哪些 campaign 花得多但 ROAS 低（左下象限）= 需要干预
- 点击气泡跳转到该 campaign 详情

## 二、发现异常 —— 「智能巡检 + 异常推送」

### 2.1 实时异常巡检仪表板

**现状**：告警中心有告警但缺乏分类整理和优先级排序。

**需求**：

```
┌─────────────────────────────────────────────┐
│  🚨 需要立即处理 (3)                          │
│  ├── Campaign A：ROAS 0.3，已花费 $50，建议暂停 │
│  ├── Campaign B：花费速率是昨日 3x，检查预算    │
│  └── 素材 #xxx：CTR 持续 5 天下降，建议换素材    │
│                                              │
│  ⚠️ 需要关注 (5)                              │
│  ├── Campaign C：ROI 较 7日均值下降 25%        │
│  └── ...                                     │
│                                              │
│  ✅ 利好信号 (2)                               │
│  ├── Campaign D：GMV 突增 150%，考虑加预算     │
│  └── 素材 #yyy：Hook Rate 35%，高于均值 2 倍   │
└─────────────────────────────────────────────┘
```

**分类逻辑**：
- 🚨 **立即处理**：ROAS < 保本线且花费 > $20 / 花费速率异常 / token 即将过期
- ⚠️ **需要关注**：指标持续下降趋势 / ROI 下降超 20% / 素材疲劳
- ✅ **利好信号**：GMV 突增 / 新素材表现优异 / ROAS 持续提升

**实现要点**：
- 后端新增 `/alerts/dashboard` 接口，按优先级分组返回
- 每条告警附带「建议动作」按钮（暂停/加预算/观察/换素材）
- 点击按钮直接执行，无需跳转

### 2.2 异常检测规则增强

**现状**：8 条默认规则，但缺少一些投放师关心的场景。

**新增规则**：

| 规则 | 触发条件 | 严重级别 |
|------|---------|---------|
| 花费速率异常 | 当前小时花费 > 日预算/24 × 2 | CRITICAL |
| 连续无转化 | 花费 > $X 且连续 N 小时 0 转化 | CRITICAL |
| 新素材冷启动超时 | 素材上线 > 3 天，展示 < 1000 | WARNING |
| ROAS 跌破保本线 | 当日 ROAS < 商品保本 ROAS | CRITICAL |
| 达人素材效果差 | 达人视频 CTR < 账户均值 50% | WARNING |
| 预算即将耗尽 | 当日花费 > 日预算 × 90% | WARNING |
| 素材表现分化 | 同 campaign 下素材 ROAS 标准差 > 均值 50% | INFO |

### 2.3 周期对比异常检测

**现状**：只做单日检测，缺少周期维度。

**需求**：
- 环比（今日 vs 昨日）
- 周同比（今日 vs 上周同天）
- 滚动均值偏离（今日 vs 过去 7 天均值）
- 后端在 `DetectionEngine` 中新增 `PeriodCompareDetector`

## 三、执行动作 —— 「一键操作 + 批量处理」

### 3.1 快捷操作中心（现有 Actions 页面重做）

**现状**：Actions 页面功能单薄。

**需求**：

```
┌─ 快捷操作 ─────────────────────────────────┐
│                                              │
│  📋 待处理队列 (7)                            │
│  ┌──────────────────────────────────────┐    │
│  │ Campaign A  ROAS: 0.3  花费: $50      │    │
│  │ AI建议: 暂停 (置信度 92%)              │    │
│  │ [✅ 执行] [❌ 忽略] [📝 自定义]        │    │
│  └──────────────────────────────────────┘    │
│  ┌──────────────────────────────────────┐    │
│  │ 素材 #xxx  Hook Rate 35%              │    │
│  │ AI建议: 加预算 $50→$100 (置信度 78%)   │    │
│  │ [✅ 执行] [❌ 忽略] [📝 改为 $80]      │    │
│  └──────────────────────────────────────┘    │
│                                              │
│  🔧 批量操作                                  │
│  [☑ 全选亏损Campaign] [暂停选中] [降预算50%]  │
│                                              │
└──────────────────────────────────────────────┘
```

**核心能力**：
- **待处理队列**：合并告警 + AI 决策，按紧急度排序
- **一键执行**：点击即调 TikTok API 执行（暂停/调预算/调出价）
- **批量操作**：勾选多个 campaign，批量暂停/降预算/启用
- **自定义参数**：AI 建议加预算到 $100，投放师可以改成 $80
- **操作日志**：每次操作记录到 `decisions` 表，标记为 `MANUAL`

### 3.2 预算调控面板

**现状**：只能单个 adgroup 调预算。

**需求**：
- 所有 campaign 的预算/花费/剩余一览
- 滑块调整预算（拖动即时预览新预算值）
- 批量设置：「所有 ROAS < 1 的 campaign 预算降 50%」
- 预算分配建议：基于历史 ROAS，推荐将预算从低效 campaign 转移到高效 campaign

### 3.3 素材操作

**现状**：可以暂停/恢复/加热单个素材。

**需求**：
- **素材替换建议**：当素材进入 DECAY/FATIGUE 阶段，推荐同类高 Hook Rate 素材替换
- **素材 A/B 对比**：选择 2 个素材，对比 Hook Rate / Hold Rate / CTR / ROAS 趋势图
- **素材淘汰规则**：设置自动规则，如「花费 > $30 且 ROAS < 0.5 则自动暂停」

## 四、验证效果 —— 「操作后追踪 + 归因分析」

### 4.1 操作效果追踪（现有 DecisionImpact 增强）

**现状**：有 `decision_impacts` 表但前端展示不完整。

**需求**：

```
┌─ 操作效果追踪 ───────────────────────────────┐
│                                               │
│  最近 7 天操作记录                              │
│  ┌───────────────────────────────────────┐    │
│  │ 3/28 15:30  暂停 Campaign A             │    │
│  │ 操作前: 花费 $50/天, ROAS 0.3           │    │
│  │ 操作后: 花费 $0/天, 节省 $50            │    │
│  │ 效果评分: ⭐⭐⭐⭐ (减少亏损 $35)        │    │
│  │ 状态: ✅ 有效                           │    │
│  └───────────────────────────────────────┘    │
│  ┌───────────────────────────────────────┐    │
│  │ 3/29 10:00  加预算 素材#yyy $50→$100    │    │
│  │ 操作前: 3单/天, ROAS 2.1               │    │
│  │ 操作后: 7单/天, ROAS 1.8               │    │
│  │ 效果评分: ⭐⭐⭐ (转化增加但ROAS微降)     │    │
│  │ 状态: ⚠️ 需要观察                       │    │
│  └───────────────────────────────────────┘    │
│                                               │
│  汇总: 7天内 12 次操作, 有效 8 次 (67%)         │
│  预估节省: $215 / 预估增收: $380               │
└───────────────────────────────────────────────┘
```

**实现要点**：
- 操作执行后自动创建 `DecisionImpact` 记录（存 before 快照）
- 24h / 48h / 72h 后自动采集 after 快照，计算 delta
- 前端展示操作时间线 + 效果评分
- 汇总统计：有效率、节省金额、增收金额

### 4.2 日报/周报

**现状**：有日报触发但内容不够丰富。

**需求**：
- 日报内容：今日花费/GMV/ROAS/利润 + vs 昨日 + 异常汇总 + 已执行操作 + 明日建议
- 周报内容：本周趋势 + 素材生命周期变化 + 最佳/最差 campaign + 操作有效率
- 格式：飞书卡片消息，关键指标带颜色标注
- 后端在 `report_task.py` 中组装富文本报告

## 五、维度丰富化 —— 「多角度分析」

### 5.1 时间维度

| 维度 | 说明 | 用途 |
|------|------|------|
| 小时级 | 每小时花费/转化曲线 | 发现「黄金时段」和「低效时段」 |
| 日级 | 每日趋势折线图 | 看整体走势 |
| 周级 | 周均值对比 | 发现周期性规律（周末 vs 工作日） |
| 月级 | 月度汇总 | 长期 ROI 评估 |

**实现**：前端加时间粒度切换（小时/天/周/月），后端按 `stat_date` 做不同粒度聚合

### 5.2 商品维度

**现状**：有 GMVMAX_ITEM 数据但展示不够。

**需求**：
- 商品排行榜：按 GMV / 订单数 / ROAS / 利润排序
- 商品生命周期：新品期 → 成长期 → 成熟期 → 衰退期
- 商品关联素材：每个商品关联了哪些创意视频，各视频表现如何
- 商品成本自动发现：从 TikTok Shop API 自动拉取商品售价，减少手动配置

### 5.3 达人/渠道维度

**现状**：有 ORDER_SOURCE 数据但前端展示不完整。

**需求**：
- 达人排行榜：按 GMV / 佣金 / 转化率排序
- 自营 vs 达人对比：花费分配、GMV 占比、ROI 差异
- 达人素材关联：每个达人发了哪些视频，各视频带货效果
- 佣金 ROI：达人佣金支出 vs 达人带来的 GMV，判断值不值

### 5.4 素材维度增强

**现状**：有 Hook Rate / Hold Rate 但缺少深度分析。

**需求**：
- 素材热力图：X 轴=投放天数，Y 轴=Hook Rate，颜色=花费量 → 一眼看出素材衰退拐点
- 素材标签体系：按内容类型（测评/种草/开箱/对比）、时长（15s/30s/60s）、风格（快节奏/慢叙述）分类
- 素材库存预警：当活跃素材 < 3 个时提醒补充
- 竞品素材参考：（手动录入竞品视频链接，对比自己的素材指标）

## 六、操作人性化 —— 「减少重复操作」

### 6.1 自动化规则引擎（现有 Rules 增强）

**现状**：有 8 条规则但只能检测告警，不能自动执行动作。

**需求**：
- 规则 = 条件 + 动作，投放师自定义
- 示例规则：
  ```
  当 Campaign 的 ROAS < 0.5 且 花费 > $30
  → 自动暂停该 Campaign
  → 发飞书通知
  ```
  ```
  当素材 Hook Rate > 30% 且 花费 < $10
  → 自动加预算 200%
  → 标记为「潜力素材」
  ```
- 前端规则编辑器：条件选择器 + 动作选择器 + 参数配置
- 规则执行日志：记录每条规则触发了多少次，效果如何

### 6.2 常用操作模板

**需求**：
- 投放师可以保存常用操作为模板
- 示例：「止损模板」= 暂停所有 ROAS < 0.5 的 campaign + 降预算 50%
- 示例：「冲量模板」= ROAS > 2 的 campaign 加预算 100% + 通知
- 一键执行模板

### 6.3 键盘快捷键

**需求**：
- `Ctrl+R` 刷新数据
- `Ctrl+P` 暂停选中的 campaign
- `Ctrl+Shift+P` 批量暂停所有亏损 campaign
- 表格行选择支持 Shift 多选

### 6.4 移动端适配

**需求**：
- 关键指标卡片在手机端可查看
- 告警推送到飞书后，点击链接可在手机端打开并执行操作
- 响应式布局：侧边栏在移动端变为底部 tab

## 七、优先级排期建议

### P0 — 立即做（解决投放师每日必做的事）
1. **今日大盘周期对比** — 投放师每天第一件事就是看今天 vs 昨天
2. **异常巡检面板** — 分类展示「立即处理/关注/利好」，附带一键操作
3. **操作效果追踪** — 操作后能看到数据变化，闭环验证
4. **批量操作** — 多选 campaign 批量暂停/调预算

### P1 — 近期做（提升效率）
5. **预算调控面板** — 所有 campaign 预算一览 + 滑块调整
6. **素材 A/B 对比** — 选 2 个素材对比趋势
7. **日报/周报增强** — 飞书富文本报告
8. **商品排行榜** — 按 GMV/利润排序

### P2 — 中期做（丰富维度）
9. **小时级数据** — 发现黄金时段
10. **达人排行榜** — 达人 ROI 分析
11. **自动化规则引擎** — 条件 + 动作自定义
12. **Campaign 矩阵气泡图** — 花费 vs ROAS 可视化

### P3 — 远期做（体验优化）
13. **操作模板** — 保存常用操作
14. **素材标签体系** — 内容分类
15. **移动端适配** — 手机可操作
16. **键盘快捷键** — 高效操作

## 八、数据流闭环示意

```
  ┌──────────────┐
  │  TikTok API  │
  │  定时同步     │
  └──────┬───────┘
         ↓
  ┌──────────────┐
  │   数据入库    │  metrics_snapshots / creatives / stores
  └──────┬───────┘
         ↓
  ┌──────────────┐     ┌──────────────┐
  │  异常检测     │────→│  告警生成     │
  │  周期对比     │     │  优先级分类   │
  └──────┬───────┘     └──────┬───────┘
         ↓                     ↓
  ┌──────────────┐     ┌──────────────┐
  │  LLM 分析    │────→│  决策建议     │
  │  多维度分析   │     │  置信度评分   │
  └──────────────┘     └──────┬───────┘
                              ↓
                       ┌──────────────┐
                       │  人工审批     │  ← 快捷操作中心
                       │  或自动执行   │
                       └──────┬───────┘
                              ↓
                       ┌──────────────┐
                       │  TikTok API  │  暂停/调预算/调出价
                       │  执行操作     │
                       └──────┬───────┘
                              ↓
                       ┌──────────────┐
                       │  效果追踪     │  24h/48h/72h 自动对比
                       │  验证闭环     │
                       └──────┬───────┘
                              ↓
                       ┌──────────────┐
                       │  日报/周报    │  汇总推送
                       │  经验沉淀     │
                       └──────────────┘
```
